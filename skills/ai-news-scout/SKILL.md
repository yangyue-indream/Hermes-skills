# AI 资讯搜集与选题发现

## 核心思路

AI 领域信息过载严重。与其被动刷信息流，不如建立一套**多源扫描 → 筛选打分 → 每日精选**的自动化管线。

## 工作流程

### 第一步：多源扫描

从以下信源并行抓取最近动态：

| 类别 | 信源 | 抓取方式 |
|------|------|---------|
| 论文 | arXiv cs.AI / cs.CL | daily-papers-fetch skill |
| 模型发布 | HuggingFace Trending | WebFetch |
| 开源项目 | GitHub Trending (AI/ML topic) | WebSearch / gh CLI |
| 行业新闻 | TechCrunch / VentureBeat AI频道 | WebFetch |
| 中文资讯 | 机器之心 / 量子位 / 36氪AI | WebFetch |
| 产品动态 | Product Hunt AI category | WebSearch |
| 社区热议 | Hacker News / Reddit r/MachineLearning | WebFetch |

### 第二步：筛选打分

对每条资讯做两轮筛选：

**第一轮：AI 相关性过滤**
- 是否直接涉及 AI/ML 技术？
- 是否是 AI 产品/工具发布？
- 是否涉及 AI 行业重要动态（融资/政策/人才）？
- 无关内容直接丢弃

**第二轮：五维打分**（每维 1-5 分）

| 维度 | 说明 |
|------|------|
| 🔥 影响力 | 对行业/用户的影响程度 |
| 🆕 新鲜度 | 是否新范式/新方法/新工具（越新越高） |
| 🛠️ 可操作性 | 能否直接应用到工作中 |
| 📈 趋势性 | 是否代表一个正在形成的趋势 |
| 🎯 相关性 | 对我们当前项目的相关程度 |

### 第三步：生成每日精选

按总分排序，输出格式：

```
📅 2026-06-24 AI 每日精选

🔴 必读 (15分+)
- [标题] [链接] | 影响力5 新鲜度4 可操4 趋势4 相关4
  一句话摘要

🟡 推荐 (12-14分)
- ...

🟢 速览 (9-11分)
- ...

💡 选题灵感
- 基于今天的信息，可以做的内容/项目方向
```

### 第四步（可选）：生成深度笔记

对必读条目，调用 paper-reader 或自行做深度分析，生成结构化笔记。

## 定制化

首次使用时，向用户确认：
1. **关注领域**：通用 AI / 大模型 / AI Agent / 开发工具 / AI 写作 / 其他
2. **深度偏好**：资讯简报 / 深度分析 / 两者都要
3. **频率**：一次性扫描 / 每日定时（用 CronCreate）
4. **输出位置**：终端显示 / 存为 Markdown / 同步到 Obsidian
